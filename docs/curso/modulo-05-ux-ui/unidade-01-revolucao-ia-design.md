# Revolução da IA no Design

## Contexto

A IA está redefinindo cada etapa do processo de design — da pesquisa com usuários à entrega de código. Ferramentas AI-driven não substituem designers, mas amplificam sua capacidade criativa e aceleram ciclos de iteração.

## AI-Driven UX/UI

O paradigma **AI-driven design** integra modelos generativos em todo o fluxo:

1. **Pesquisa** — análise de feedback com [[nlp]], clustering de insights
2. **Ideação** — geração de variações visuais, moodboards automáticos
3. **Prototipação** — [[text-to-ui]], wireframes a partir de descrições
4. **Implementação** — design-to-code, componentes gerados
5. **Validação** — testes A/B automatizados, heatmaps preditivos

## Ferramentas por Etapa do Design

| Etapa | Ferramentas | Capacidade IA |
|-------|-------------|---------------|
| Pesquisa | Maze AI, Hotjar AI | Síntese de entrevistas, padrões de uso |
| Ideação | Midjourney, DALL-E, Galileo AI | Geração visual, exploração criativa |
| Wireframe | Uizard, Relume | Texto → wireframe estruturado |
| Protótipo | [[figma]] + plugins IA, Firebase Studio | Layout automático, variações |
| UI Code | v0.dev, Bolt, Lovable | [[text-to-ui]] → código React |
| Testes | Testim, Applitools | Testes visuais com IA |

## Casos de Uso Reais

### 1. Design System Acceleration
Usar IA para gerar variações de componentes dentro de um [[design-system]] existente, mantendo consistência de tokens.

### 2. Personalização de UI
Interfaces que se adaptam ao comportamento do usuário usando [[ml]] para prever preferências de layout e conteúdo.

### 3. Acessibilidade Automatizada
Ferramentas que detectam problemas de contraste, hierarquia e navegação por teclado automaticamente.

## Impacto no Fluxo de Trabalho

```
Antes:  Pesquisa → Wireframe → Protótipo → Dev → Testes (semanas)
Depois: Prompt → Protótipo funcional → Iteração → Deploy (horas/dias)
```

O ciclo de feedback encurta drasticamente, permitindo mais experimentação com menos custo.

## Limitações e Cuidados

- IA gera **opções**, não **decisões** — o designer curadoria
- Consistência de [[design-system]] requer supervisão humana
- Acessibilidade gerada por IA precisa de validação manual
- Viés nos dados de treinamento pode gerar padrões excludentes
