# Live Coding

## O que Avaliam

| Dimensão | Peso | O que observam |
|----------|------|----------------|
| Comunicação | 30% | Pensa em voz alta, pergunta, clarifica |
| Problem-solving | 30% | Abordagem estruturada, edge cases |
| Código | 25% | Limpo, funcional, idiomático |
| Testes | 15% | Valida solução, considera falhas |

## Framework de Resolução

```
1. ENTENDER (2-3 min)
   - Repetir o problema com suas palavras
   - Perguntar sobre constraints e edge cases
   - Confirmar input/output esperado

2. PLANEJAR (3-5 min)
   - Pensar em voz alta
   - Considerar 2-3 abordagens
   - Escolher e justificar

3. IMPLEMENTAR (15-20 min)
   - Código limpo desde o início
   - Narrar enquanto escreve
   - Não otimizar prematuramente

4. TESTAR (3-5 min)
   - Rodar com exemplo do enunciado
   - Testar edge cases
   - Analisar complexidade (tempo/espaço)
```

## Dicas Específicas para AI Engineer

- Problemas podem envolver processamento de texto/NLP
- Familiarize-se com manipulação de embeddings (arrays, distâncias)
- Saiba implementar busca por similaridade (cosine, dot product)
- Conheça estruturas para grafos (agentes, workflows)

## Anti-padrões

- ❌ Ficar em silêncio por minutos
- ❌ Começar a codar sem entender o problema
- ❌ Ignorar hints do entrevistador
- ❌ Desistir ao travar (peça dica, é esperado)

## Preparação

- LeetCode/HackerRank: 2-3 problemas/dia por 4 semanas
- Foco em: strings, arrays, hashmaps, trees, graphs
- Praticar com timer (30-45 min por problema)
- Simular com amigo (pair programming style)
